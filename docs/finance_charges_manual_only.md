# חיובים נוספים — רק הוספה ידנית

## התנהגות נוכחית

רשומות ב־`extra_charges` של לקוח נוצרות **רק** כשמוסיפים חיוב דרך מסך הכספים / טופס עדכון כספים (למשל `update_finance` עם פעולת `extra`, או נתיבי ה־API הרלוונטיים ב־`finance`).

## מה הוסר

בעבר, בעת סימון משימה כ־**הושלם**, השרת (`app.py`) הוסיף אוטומטית חיובים עם כותרות כמו:

- `מחיאות כפיים - <שם משימה>` (סכום קבוע 50)
- `👏 <שם משימה>` (סכום 0, בזרימת עדכון משימה אחרת)

הלוגיקה הזו **בוטלה**. השלמת משימה לא יוצרת יותר חיובים.

## ניקוי נתונים

חיובים קיימים שכותרתם התחילה ב־`מחיאות כפיים` או ב־`👏 ` הוסרו מקובץ `agency_db.json` (סקריפט חד־פעמי). אם יש לך עותקים נוספים של מסד הנתונים (גיבויים, סביבות אחרות), יש להריץ שם ניקוי דומה אם צריך.
